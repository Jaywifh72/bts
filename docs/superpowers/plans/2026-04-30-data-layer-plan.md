# Data Layer Implementation Plan — Global Cinematic Technical Repository (v1)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v1 Postgres data layer for the Global Cinematic Technical Repository — schema, migrations, seed data, killer-query and cascade tests — as a `packages/db` workspace ready to be consumed by sub-project 2 (the public web app).

**Architecture:** A pnpm-workspace monorepo with one v1 package: `packages/db`. PostgreSQL 16 in Docker locally, Drizzle ORM for schema-as-typescript, drizzle-kit for migrations, Vitest for the three test suites (migration sanity, cascade matrix, killer queries). All schema decisions, conventions, and seed selection are pinned in the spec at `docs/superpowers/specs/2026-04-30-data-layer-design.md`.

**Tech Stack:** TypeScript (strict), PostgreSQL 16, Drizzle ORM + drizzle-kit, postgres.js, Zod (for JSONB spec validation), Vitest, Docker Compose, pnpm.

**Prerequisite reading:** `docs/superpowers/specs/2026-04-30-data-layer-design.md` — every schema decision in this plan is anchored there.

**Out-of-scope (per spec §9):** TMDb/IMDb sync, full-text search, auth, web app, admin UI, asset modeling, advanced lens/sensor/emulsion tables.

- [ ] **Step 3: Create `packages/db/drizzle.config.ts`**

```ts
import 'dotenv/config';
import { defineConfig } from 'drizzle-kit';

if (!process.env.DATABASE_URL) {
  throw new Error('DATABASE_URL is required');
}

export default defineConfig({
  schema: './src/schema/!(index).ts',
  out: './migrations',
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL,
  },
  strict: true,
  verbose: true,
});
```

**Note on `schema` path:** drizzle-kit v0.30 uses a CJS `require()` loader that does NOT understand NodeNext ESM `.js` extensions in TypeScript re-exports. If `schema` points at `index.ts` and that file does `export * from './enums.js'`, drizzle-kit tries to resolve a literal `enums.js` file and fails. The glob `'./src/schema/!(index).ts'` matches every schema file *except* the index, sidestepping the issue while letting runtime code use the standard NodeNext re-export pattern.

# Phase B: Independent schemas (Tasks 4–7)

Each task in this phase creates a schema file, generates a migration, applies it, and asserts the resulting tables behave correctly. The pattern across all four tasks is:

1. Write the Drizzle schema.
2. Re-export from `src/schema/index.ts`.
3. Run `pnpm --filter @bts/db generate` → drizzle-kit produces a SQL migration in `migrations/`.
4. Run `pnpm --filter @bts/db migrate` → applies to the dev DB.
5. Smoke test (insert + select + assert) using the Drizzle client.
6. Commit migration + schema together.

**Why `--custom`:** Drizzle's migration generator doesn't model triggers, so the SQL has to be written by hand. **However**, manually creating a `.sql` file in `migrations/` does *not* register it in `migrations/meta/_journal.json`, and `drizzle-kit migrate` only runs files that are journaled. The supported workflow is `drizzle-kit generate --custom`, which scaffolds an empty migration file *with the journal entry already added* — you then fill in the SQL.

- [ ] **Step 2: Fill in the trigger SQL**

Open the new `<NNNN>_updated_at_triggers.sql` file and replace its contents with:

```sql
-- Function: set updated_at to now() on UPDATE
CREATE OR REPLACE FUNCTION set_updated_at() RETURNS trigger AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
--> statement-breakpoint
-- Attach to every table that has updated_at
DO $$
DECLARE
  r record;
BEGIN
  FOR r IN
    SELECT table_name
    FROM information_schema.columns
    WHERE table_schema = 'public'
      AND column_name = 'updated_at'
  LOOP
    EXECUTE format(
      'DROP TRIGGER IF EXISTS set_updated_at_trigger ON %I;', r.table_name
    );
    EXECUTE format(
      'CREATE TRIGGER set_updated_at_trigger BEFORE UPDATE ON %I FOR EACH ROW EXECUTE FUNCTION set_updated_at();',
      r.table_name
    );
  END LOOP;
END$$;
```
